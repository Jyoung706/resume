## 2. 신뢰성 / 과장 리스크 (면접에서 깨질 지점)

**판정: 가장 위험한 영역. "단독/주도" 인플레이션과 무근거 수치.**

### 2-1. "단독 / 주도"의 남발
이력서 전반에 단독·주도·전반·단일이 도배돼 있다. 증거 자료(00) 대조 결과:

| 이력서 표현 | 증거상 실제 | 리스크 |
|---|---|---|
| k8s 플랫폼 **"단독"** 구축 | 98/123커밋(~80%), seoyujin이 ai/preprocess 매니페스트 일부 | **"단독" → "주도(80%)"로 정정 권장.** 면접에서 "전부 혼자?" 물으면 무너짐 |
| 프레임워크/백엔드 "주도" | 305커밋, 32,906 LOC → **정당함** | 안전 |
| orkis-jobs "단독 구축" | 68커밋, 단독에 가까움 → 대체로 OK | 안전 |
| (Tech Skills에 LangGraph, RAG) | orkis-ai 25커밋=배포만, preprocess는 본인 0커밋 | RAG/AI 파이프라인 "내가 만들었다"로 오해되면 위험 |

**핵심 경고**: 증거 자료 스스로가 "orkis-interface 과장 금지", "preprocess 본인 작업 아님", "orkis-ai 앱코드 0"이라고 못박았다. 이력서엔 직접 위반은 없지만, **About의 "시스템을 처음부터 끝까지" + Skills의 AI/LLM 나열**이 합쳐지면 "AI 플랫폼 전체를 혼자 만든 사람"이라는 과대 인상을 준다. 면접관이 "Text-to-SQL의 SQL 생성/RAG 모델 부분 설명해보라"고 하면 "그건 연구원이…"가 나온다. **선 긋기를 이력서에서 먼저 해라.**

### 2-2. 무근거 / 검증 불가 수치
- **"동시 채팅 처리량 개선"(orkis-desktop-ai)** — 기준선도, 개선폭도 없음. 측정 안 한 정황. **가장 약한 한 줄.** 숫자 없으면 차라리 "asyncio.to_thread 오프로드로 이벤트 루프 블로킹 제거" 같은 정성 표현으로.
- **"오버헤드 최소화"(Lazy Proxy)** — 무엇 대비 얼마? 벤치마크 없음. "2+depth Bean만 clone"이라는 구체 메커니즘은 좋으나 "최소화"는 입증 책임을 남긴다.
- **LOC를 성과로 제시**(7,658 / 32,906 / +14,131 LOC). LOC는 시니어에게 **마이너스 신호**(많이 짠 게 자랑 아님). 커밋수·LOC는 "규모 입증"용으로 1~2회만, 성과 문장에선 빼라.
- **"1,000줄+ 제거"** — 이건 긍정(부채 감소)이라 OK. 유지.

### 2-3. 모호어
"전반을 주도", "처음부터 끝까지", "시스템을 연결" — 멋있지만 면접관에겐 "그래서 측정 가능한 결과가 뭔데?"를 부른다.

## 3. 경력직 적합성 (임팩트 / 오너십 / 협업)

**판정: 오너십·기술깊이 = 상 / 비즈니스 임팩트 = 하 / 협업 증거 = 하**

- **오너십**: 강하다. 0→1로 프레임워크+앱+데스크탑+AI백엔드+k8s를 세운 건 3년차 평균을 상회. 이건 진짜 강점.
- **비즈니스 임팩트: 거의 없음(최대 약점).** 전 항목이 "무엇을 어떻게 만들었다"의 **기능/구현 나열**이다. 다음이 전부 빠져 있다:
  - 사용자 수 / DAU / 처리 트래픽 / RPS / 데이터 규모
  - 비용 절감(예: 노드풀 분리로 리소스 X% 절감, 백업 자동화로 운영시간 Y 절감)
  - 장애·다운타임 감소, SLA, 배포 빈도/리드타임 변화
  - "이 시스템이 회사/연구과제에 어떤 결과를 만들었나"
  - → **연구소라서 실사용자 임팩트가 구조적으로 없다**는 게 핵심 딜레마. 그렇다면 **"개발 생산성/운영 효율" 임팩트라도 수치화**해야 한다. (예: "Jenkins 동적 에이전트로 빌드 동시성 확보, 배포 N분→M분", "GitOps 전환으로 환경 재현 가능")
- **협업 증거 부족**: "AI 연구원·프론트 개발자와 협업"이 이전본엔 있었는데 v2 본문에선 거의 사라졌다. **팀 규모, 본인 롤(리드/단독), 코드리뷰, 의사결정 방식**이 안 보인다. 3년차 경력직 평가에서 "혼자 다 했다"는 양날—협업 못 하는 사람으로도 읽힌다.

**기능 나열 vs 임팩트**: 현재는 70% 기능 나열. "구현했다"를 "왜 했고(문제), 결과가 뭐였나(임팩트)"로 바꿔야 경력직 문서가 된다.

## 4. 차별점 / 평범한 지점

**강점(차별)**
- **k8s 플랫폼을 매니페스트로 단독 80% 구축 + DR/백업까지** — 3년차에서 드묾. 특히 "토큰 디스크 미저장 미러 백업", "cert-manager TLS 자동화", "Jenkins 동적 에이전트 파드"는 구체적이고 검증 가능. **여기를 전면에 더 세워라.**
- **온프레미스(E3TS)→클라우드 네이티브(범익) 서사** — 인프라 직군에 설득력 있는 성장 스토리.

**평범하거나 위험한 지점**
- **자체 프레임워크 = 양날의 검.**
  - 긍정: DI/트랜잭션 전파/Proxy 메타프로그래밍을 직접 구현 → "원리를 안다"는 강력한 신호. 깊이 면접에 유리.
  - 부정: 시니어 다수는 **"왜 Nest/TypeORM 두고 305커밋짜리 자체 프레임워크를? 버스 팩터 1, 유지보수 부채, NIH, 오버엔지니어링 아닌가?"** 라고 즉시 의심한다. 연구소라 가능했던 사치로 본다.
  - **방어 논리를 이력서 안에 넣어라**: 예) "Nest는 OOO 제약(예: 멀티테넌시 동적 커넥션/멀티DB 추상화 요구)으로 부적합 판단 → 핵심만 자체 구현." 또는 "학습·검증 목적과 제품 요구가 겹쳐…". **선택의 정당성**이 없으면 강점이 약점으로 뒤집힌다. (대표 성과 1번에 한 줄 추가가 최선)
- **AI/LLM 스킬**: 연동·SSE 수준인데 LangGraph까지 나열 → 깊이 질문 들어오면 얕음 노출. 솔직 수준으로 톤다운.

## 5. 누락 / 채용담당자가 의심할 지점

1. **실서비스 트래픽 부재** — 전 경력이 연구소+SI성. "프로덕션 대규모 운영 경험 0"이 명백. 대규모 트래픽 회사는 여기서 거른다.
2. **테스트 / 품질 증거 전무** — 단위·통합 테스트, 커버리지, CI에서의 테스트 게이트 언급이 **한 줄도 없다.** 자체 프레임워크를 만들었다면서 테스트가 없으면 "검증 안 된 코드"로 본다. **치명적 누락.**
3. **관측성(Observability) 빈약** — metrics-server/OpenLens만. 로깅 수집(ELK/Loki), 메트릭(Prometheus/Grafana), 알림, 트레이싱 부재. SRE/인프라 직군 면접관이 반드시 짚는다.
4. **이직 사유 / 짧은 근속** — E3TS 1년 2개월. 짧다. 그리고 2022.10 부트캠프 수료~2023.06 입사 사이 **약 8개월 공백** 미설명.
5. **팀 규모·본인 포지션** — 연구소 인원, 본인이 인프라 단독 담당인지, 리뷰어가 있었는지 불명. "혼자 다 함"이 협업 부재 신호로 해석될 여지.
6. **포트폴리오/실물 링크** — v2엔 포트폴리오 링크가 빠졌다(이전본엔 있었음). 자체 프레임워크 같은 주장은 코드/문서 링크가 있어야 신뢰가 산다.

## 6. 구체적 수정 제안 (우선순위)

### P0 — 통과/탈락을 가르는 것 (반드시)
- **[P0-1] "단독" 표현 정정.** k8s "단독" → "주도(약 80%, 동료와 분담)". 과장 리스크 1순위 제거. (line 17, 75, 117)
- **[P0-2] 비즈니스/운영 임팩트 수치 1~2개 확보.** 연구소라 사용자 임팩트가 없다면 **운영 효율 임팩트**라도: 배포 리드타임 단축, 환경 재현성, 백업 자동화로 절감된 운영시간, 노드풀 분리로 리소스 효율 등. 측정 안 했으면 지금이라도 추정·기록.
- **[P0-3] 테스트/검증 한 줄 추가.** 자체 프레임워크·트랜잭션의 정확성을 어떻게 보장했는지(테스트 유무). 없으면 "xmin 기반 격리 검증" 같은 실제 검증 행위를 "테스트/검증" 섹션으로 명시.
- **[P0-4] 자체 프레임워크 방어 논리 1줄.** 대표 성과 1번에 "Nest 대신 자체 구현한 이유(멀티테넌시 동적 커넥션·멀티DB 요구)"를 넣어 NIH 의심 차단.
- **[P0-5] 무근거 수치 제거/정성화.** "동시 채팅 처리량 개선"→측정값 넣거나 메커니즘 표현으로. LOC를 성과 근거로 쓰는 문장 정리.
