# 증거 보강 (v3용) — 사용자 피드백 3건 + 추가 submodule

> 1차 증거(00)에 더해, 사용자 피드백으로 추가 분석한 내용. 경력기술서 작성에도 이 파일 사용.

## A. orkis-backend 채팅 파이프라인 — 본인 비주도 (코드 구조 교차검증)

- orkis-backend는 GitHub 스냅샷 1커밋 → 커밋 단위 작성자 구분 불가. **코드 구조로 판정.**
- **본인 주도 확정**(파일 근거):
  - OAuth: `src/main/auth/AuthService.ts`(530줄) — 네이버/카카오/구글 통합, parseOAuthUserInfo 제공사별 매핑
  - 동적 DB: `src/main/db-connection/DbConnectionService.ts` extends `DynamicConnectionSupport` — 사용자별 DB 등록, **AES-256-GCM 암호화**(encryptPassword/decryptPassword), PG/SQLite/MySQL·MariaDB 타입별 분기
  - 단일 코드베이스 빌드: `tsconfig.desktop.json` paths `@/*:[src/desktop, src/main]` fallback + `tsc-alias`, `build:desktop` 스크립트. desktop override vs cloud 재사용을 tsconfig 주석으로 명시
  - 배포: `Dockerfile`/`Dockerfile.desktop`, k8s 매니페스트
- **채팅 파이프라인 = 타 개발자**(판정):
  - 클라우드 `src/main/sse/services/FetchStreamService.ts`(819줄, Redis XREAD 폴링) → 직접 증거 없음, tsconfig desktop **exclude 목록**에 포함 → 타 개발자 영역
  - 데스크탑 `src/desktop/chat/ChatStreamService.ts`(753줄, AI SSE 직접 구독) → tsconfig 주석 "desktop 따로 구현" → 본인의 **"데스크탑 어댑테이션"**(파이프라인 설계 아님)
- **이력서 반영**: 채팅/RAG SSE 스트리밍을 주도 성과에서 제외. 단일 코드베이스 맥락의 "데스크탑 어댑테이션(프론트 무수정 동일 이벤트 계약)"으로만 1줄.

## B. orkis-desktop-ai — 클라우드 강결합 디커플링·이식 (54커밋)

- **맥락**: 기존 클라우드 AI(orkis-ai)가 Ray·Redis 강결합 → 데스크탑 단일 머신에서 동작하도록 분리. 비즈니스 로직(LangGraph)은 원본 재사용. 새 언어(Python) 학습·적용. **Python 숙련 주장 아님 → Tech Skills에서 Python/FastAPI 제외.**
- **디커플링 대조**(파일 근거):
  - Ray Actor 6종(ConversationActor/PreprocessActor/LLMActor/EmbeddingActor/FaissActor 등, `ray_config.py` 예약메모리 합 3,122MB) → `core/task_manager/chat.py` asyncio Task + `BaseTaskManager` ABC, `LocalLLMActor._RemoteCallable`로 `.remote()`를 로컬 async로 변환
  - Redis XADD 스트림 → `app/transport/sse_transport.py` asyncio.Queue + `EventTransport` ABC(`core/transport/base.py`) + `AsyncTransportStreamHandler`
  - PostgreSQL/Redis → SQLite/파일(`app/preprocess/history_dao.py`, vector_search sqlite repository)
  - 커밋: 6f50b54(태스크매니저), d2f3bcd(TCP socket 제거→SSE), 64a2080(SQLite 직접 UPDATE), 482c21d(cloud FaissPreprocessService 재사용), 999ce64(Transport ABC), 6b52d98(gpt-5 풀 오염 버그)
- **메모리/운영 개선 추정(소스 기반, 실측 아님)**:
  - orkis-ai 추정 상주: Ray Actor 예약 3,122MB + Ray 런타임(GCS/object store) 500~800MB + Redis 100~300MB ≈ **4.0~5.5GB**
  - orkis-desktop-ai 추정: uvicorn ~150MB + TaskManager/Queue/httpx ≈ **1.5~2.5GB**
  - **절감 ≈ 60%(4.5→2.0GB 기준)**. 프로세스: Ray head+N워커+Redis → 단일 uvicorn. 포트/배포/디버깅 단순화.
  - 근거: `ray_config.py` MEMORY_MB 설정값 + Ray/Redis 일반 메모리 특성. **런타임 패턴에 따라 변동.**
- **고급 패턴(역량 입증)**: pre-cancel placeholder(레이스 처리), ContextVar로 to_thread 워커 컨텍스트 주입, FastAPI lifespan+SSE StreamingResponse, gpt-5 캐시키 오염 근본원인 진단.

## C. E3TS 참고 레포 3종 (완전한 git 이력, 본인 메인 author: Jeon/luke.jeon@e3ts.co.kr·jYoung706)

> E3TS 보강용. 합산 ~1,100커밋. 경력기술서에서 상세 활용.

### parking-server — 발렛파킹 주차관제 백엔드 (385커밋)
- 스택: Express 4 + Socket.io 4 + PM2 + MySQL2/TypeORM 0.3 + JWT + bcrypt + helmet + web-push
- 규모: REST **119 엔드포인트**, Controller 21/Service 21/DAO 22(3계층 Layered), JS 83파일
- 핵심: `utils/socket.js` JWT 쿠키 인증 소켓, **중복 로그인 감지→기존 연결 강제 종료**(connectedUsers 맵), HTTP·WebSocket 동일 비즈니스 로직 통합, TypeORM 다중 데이터소스(main+SMS), Web Push
- 커밋 흔적: polling→websocket transport 전환, 주차 이력 실시간 반영

### cluster-chat-server — 그룹웨어 채팅 서버 (309커밋)
- 스택: Express 4(ESM) + Socket.io 4 + **@socket.io/redis-adapter + ioredis** + Mongoose(MongoDB) + MySQL2 + node-cron + PM2
- 규모: REST 48, **Socket 이벤트 8종**(signIn/sendChat/joinRoom/leaveRoom/leaveAllRooms/readChat/exitRoom/inviteRoom), JS 84파일
- 핵심:
  - **PM2 cluster** `ecosystem.config.cjs`(instances=CPU/2, max_memory_restart 1G, autorestart)
  - **Redis 어댑터 클러스터 브로드캐스트** `redis.connection.js`(IoRedis.Cluster pub/sub → io.adapter)
  - **namespace 멀티테넌시** `io.of('/'+companyId)` 회사별 격리 + userId/companyId 이중 인증
  - **분산 락** `distributeLock.js`(NODE_APP_INSTANCE, Redis key expire 데드락 방지)로 cron 중복 실행 방지
  - **JWT 만료 자동 갱신**(TokenExpiredError→refreshTokenCheck→재인증)
  - MongoDB(메시지)+MySQL(사용자/회사) 하이브리드, namespace별 독립 Mongo 연결

### cluster-chat-client — 채팅 클라이언트 (406커밋)
- 스택: Vue 3.4 + Vue Router 4 + **Pinia(persist)** + socket.io-client 4 + **Vite 5 + Electron 31 + electron-builder 24** + Kendo UI + Tailwind
- 규모: Vue 컴포넌트 86, JS 28
- 핵심:
  - **Pinia `socketStore`** 다중 namespace 동시 연결(Promise.all 병렬 init), 9종 수신 이벤트 콜백 레지스트리
  - **Electron** `electron/main.js`(contextIsolation, IPC, 다운로드 폴더 자동생성, **파일명 충돌 회피 알고리즘** getUniqueFilePath, 시스템 트레이, env.json 동적로딩)
  - **electron-builder** mac(dmg)/Windows(NSIS, 시작프로그램 레지스트리) 크로스플랫폼, `reload:build` 웹훅으로 배포 시 클라이언트 자동 갱신
  - GitLab CI(.gitlab-ci.yml) 브랜치별 빌드
