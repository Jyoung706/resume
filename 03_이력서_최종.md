### 1. 필요한 개발 환경을 스스로 만듭니다 — 온프레미스에서 클라우드 네이티브로

E3TS 입사 당시 개발 2명이 ssh-remote로 한 폴더를 공유해 작업하다, 인력이 늘며 충돌이 잦아졌습니다. 대표·팀장께 사내 Git 도입을 먼저 건의하고 GitLab CE·Jenkins를 서버에 단독으로 구축했으며, 도메인·SSL까지 적용한 뒤 Git을 모르는 팀원에게 컨벤션과 Git flow를 직접 교육하며 정착을 이끌었습니다.

범익 연구소에서는 이 경험을 한 단계 끌어올려, KT Cloud 쿠버네티스 위에 클러스터·네트워킹·CI/CD·백업까지 인프라 전체를 코드로 구성(GitOps)했습니다. "환경은 직접 만든다"는 태도가 온프레미스에서 클라우드 네이티브로 자연스럽게 이어졌습니다.

### 2. 모르는 분야도 끝까지 파고들어 해결합니다

E3TS에서는 네트워크·메일 프로토콜 지식이 전무한 채로 그룹웨어 메일 부문을 맡았습니다. SSL 연동 에러가 검색으로 풀리지 않자 1~2주간 로그와 수·발신 서버 config의 주석까지 읽어 설정 한 줄을 찾아 해결했고, Gmail 발신 문제는 SPF·DKIM·DMARC로, 보안은 WAF(ModSecurity)로 대응했습니다.

범익에서는 더 깊은 문제를 만났습니다. 에러조차 없이 빈 화면·메모리 누수로만 드러나는 "조용한 버그"를, 의존 라이브러리와 DB 드라이버 내부 동작까지 추적해 근본 원인을 찾아 해결했습니다. _(SQLite 동시성·프레임워크 설계 등 상세 사례는 포트폴리오 참고)_

### 3. 기술을 사람·비즈니스와 연결합니다

E3TS에서 그룹웨어 채팅을 풀스택으로 단독으로 맡으며, 신입임에도 매주 고객사 미팅에 담당자로 참석해 개발 현황 보고와 기능·공수 조율을 했습니다. 인원이 적어 각자가 PM처럼 일한 환경 덕에 "이 기능이 왜 필요한가"를 비즈니스 언어로 고민하는 습관이 생겼습니다.

범익에서는 AI 연구원·프론트엔드 개발자와 협업하며 서버를 양쪽의 접점으로 설계했습니다. 프론트 코드를 바꾸지 않고도 데스크탑·클라우드를 호환시키고, AI와 이벤트 계약을 맞추는 방식으로 협업의 마찰을 줄였습니다.
