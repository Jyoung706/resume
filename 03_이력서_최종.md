## Introduce.

### 1. 사내 형상관리 도입으로 개발 환경 개선

입사 당시 사내 서버 환경은 온프레미스로, 작게 구축된 서버실이 있었습니다. 개발 인원이 2명이라 VS Code 확장 ssh-remote로 하나의 프로젝트 폴더에 여러 명이 접속해 작업했습니다. 하지만 그룹웨어 프로젝트를 수주하고 인력을 충원하면서, 여러 명이 동시에 원격 서버로 작업하다 보니 충돌과 이슈가 잦아졌습니다.

이를 개선하고자 대표님·팀장님께 사내 Git 도입을 적극적으로 건의했고, 보안을 위해 서버실의 서버 하나를 Git 서버로 사용하기로 하여 GitLab CE와 Jenkins를 A부터 Z까지 혼자 설치·구축했습니다. 메일 서버를 담당하며 네임 서버를 다룬 경험이 있어 도메인을 적용하고 SSL 인증서로 HTTPS도 적용했습니다.

이후 Git을 잘 모르는 팀원들을 위해 기본 컨벤션과 Git flow를 직접 설명·교육하며 도입을 이끌었습니다.

### 2. 이슈 해결 능력 및 오픈소스 활용

메일 관련 네트워크·프로토콜 지식이 전무한 상태에서 그룹웨어의 메일 부문을 담당했습니다. 걱정이 많았지만 블로그·공식 문서·해외 자료를 폭넓게 참고하며 진행했습니다.

Postfix·Dovecot 오픈소스로 메일 서버를 구축하고, 그룹웨어 메일 클라이언트와 연동하기 위해 SSL/TLS 보안 연결이 필요해 네임서버·도메인까지 학습했습니다. SSL 적용 후 마주한 연동 에러는 구글링으로 해결되지 않아, 약 1~2주간 로그 분석과 수·발신 서버 config의 주석까지 꼼꼼히 읽은 끝에 설정 하나를 바꿔 정상 동작시켰습니다.

또한 사내 메일이 Gmail로 발신되지 않는 이슈를 SPF·DKIM·DMARC 정책을 메일/네임 서버에 적용(opendkim·opendmarc)해 해결했고, 보안을 위해 Nginx에 오픈소스 WAF인 ModSecurity·CRS를 적용해 SQL Injection·XSS에 대비했습니다.

연구소로 자리를 옮긴 뒤에는 더 깊은 문제들을 만났습니다. SQLite의 동시 트랜잭션 충돌을 비동기 mutex로 직렬화하고, `sqlite3` 드라이버의 RETURNING 더블 실행 버그를 스트리밍 방식으로 우회하는 등 **의존 라이브러리·DB 드라이버 내부 동작까지 추적해 문제를 해결**했습니다. 오픈소스 활용을 두려워하지 않고 오히려 즐겁게 적용해 왔습니다.

### 3. 의사소통 능력이 뛰어난 개발자

그룹웨어 채팅 파트를 풀스택으로 단독으로 맡아, 매주 월요일 고객사 미팅에 프로젝트 담당자로 참석해 개발 현황을 보고하고 필요한 기능·공수를 함께 논의했습니다. '신입이 왜 고객사 미팅에?' 싶을 수 있지만, 인원이 적은 중소기업에서는 구성원 각자가 PM처럼 일해야 했습니다. 이 과정에서 결국 개발자에게 의사소통이 중요하다는 것을 몸소 체험했습니다.

디자이너·퍼블리셔·프론트엔드 개발자와 협업하면서도 갈등 없이 소통한 경험이 있어, 어떤 사람들과도 문제없이 협업할 수 있다고 자신합니다.

### 4. 개발자의 직업에 대한 이해

개발자는 단순히 코드로 소프트웨어를 생산하는 직업이 아니라, **비즈니스·마케팅에 대한 이해, 다양한 직군과의 협업, 문제 해결 능력** 이 세 가지가 가장 중요한 덕목이라고 생각합니다.

비즈니스를 이해해야 충성 고객을 모으는 기능을, 마케팅을 이해해야 소비자를 끌어모으는 매력 포인트를 만들 수 있습니다. 또한 여러 직군과의 협업에서 최상의 결과물을 도출해야 하므로 협업 능력이 중요합니다.

마지막으로 문제 해결 능력이 가장 중요하다고 생각합니다. 요즘은 AI가 코드 작성을 많이 도와 코딩 난이도가 낮아졌지만, 이슈는 각자의 상황이 모두 다르기에, 발생한 문제를 어떻게 판단하고 **리스크가 가장 낮은 방향으로 해결할지**가 개발자의 큰 덕목이라 믿습니다. 실제로 에러 없이 빈 화면·메모리 누수로 나타나는 '조용한 버그'들을 근본 원인까지 추적해 해결해 온 경험이 그 믿음을 뒷받침합니다.

주니어 개발자이지만 개발에 대한 진심을 담아 항상 공부하고, IT 업계에서 일하는 것을 감사히 여기며 즐겁게 일하고 있습니다. 앞으로 귀사와 함께한다면 이러한 열정으로 꼭 보탬이 되겠습니다. 긴 글 읽어주셔서 감사합니다.
