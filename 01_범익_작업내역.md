## 0. 프로젝트 개요 — Orkis

자연어 질문을 입력하면 **LLM/RAG 서버가 SQL을 생성하고, 사용자가 등록한 DB에 실제로 실행해 결과를 반환**하는 Text-to-SQL 연구 플랫폼.

클라우드(웹) 버전과 데스크탑(단일 머신) 버전을 **하나의 코드베이스**로 동시 지원하며, 7개 저장소로 구성됨.

## 1. @orkis/core — 자체 백엔드 프레임워크 설계·구현 (285 커밋)

Node.js/TypeScript 환경에서 **Spring Boot 스타일의 데코레이터 기반 개발**을 가능하게 하는 프레임워크를 직접 설계.

- **DI 컨테이너**: `@Controller`/`@Service`/`@Component`/`@Dao`/`@Autowired` 데코레이터, ComponentScan 자동 빈 등록. 메타데이터 저장(데코레이터)과 인스턴스화·주입(BeanResolver)을 책임 분리. *(`0311513`, `8840f6a`)*
- **선언적 트랜잭션(`@Transactional`)**: 메서드 래핑 기반 실행, **전파 정책(REQUIRED / REQUIRES_NEW / SUPPORTS)** 지원. 트랜잭션 컨텍스트를 Bean clone + Lazy Proxy로 격리·전파. *(`d5cf28b`, `4d474d2`, `5474a40`)*
- **멀티 DB 추상화**: `BaseAdapter` 하나로 PostgreSQL·SQLite·MariaDB·Redis·File을 통일. `supportsTransaction()`로 어댑터별 트랜잭션 지원 분기. *(`a709638`, `9bff5f3`, `0da5e07`)*
- **동적 DB 연결(멀티테넌시)**: 요청의 userId로 사용자별 DB 정보를 조회해 런타임에 connection을 동적 생성 — Text-to-SQL의 핵심 인프라. *(`ffad62c`, `a1b5ebe`)*
- **SSE**: `@SSEConnection` 데코레이터 + `SSEStream` 유틸 (멀티라인 직렬화·flush·onClose). *(`2a15f51`)*
- **HTTP 클라이언트**: undici 기반 keep-alive 연결 풀링 (Agent / Pool dispatcher). *(`e74fee5`)*
- **인터셉터·운영 기능**: path 패턴 매칭 인터셉터, 파일 업로드, Unix Domain Socket, Redis 세션 플러그인, CORS·로깅·글로벌 에러 핸들러, k8s 환경 대응.

## 2. orkis-backend — 백엔드 애플리케이션 (327 커밋)

- **OAuth 소셜 로그인**: 네이버·카카오·구글을 단일 인터페이스(`OAuthConfig` 맵 + `processLogin`)로 통합. authorization code → token → userInfo 정규화 → 신규/기존 분기. bcrypt 패스워드 로그인 병행, Bearer 토큰 인증. *(`dd343b1`)*
- **동적 DB 연결 적용**: 코어의 `prepareDynamicDBConnection`을 백엔드에 적용, 등록된 연결정보 복호화 → 동적 어댑터 주입 → 사용자 DB liveness 측정(`DbConnectionChecker`). *(`657741c`, `47b09af`)*
- **RAG/LLM 챗 SSE 스트리밍**: AI(RAG)가 Redis 스트림에 쓰면 폴링 → `message_stream` 이벤트 변환 → 프론트 SSE. 토큰 스트리밍·step stepper·제목 생성·SQL 실행·취소 전 구간 처리.
- **세션 Redis 전환**: 세션 스토어를 Redis로 분리(`SessionPlugin`), 멀티 Pod 세션 공유. *(`c36963e`)*
- **단일 코드베이스 cloud/desktop 빌드 아키텍처**: TS `paths` fallback(`@/* → [desktop, main]`) + `tsc-alias`로 같은 import가 빌드 타깃에 따라 desktop override 우선 / main fallback으로 해석되게 설계. *(`e84b300`, `5c7c6b9`, `32c8a42`)*
- **인프라/배포**: Dockerfile 멀티스테이지, k8s 무중단 롤링 업데이트(`maxSurge:1/maxUnavailable:0`), PM2 클러스터 + PG Pool 검증, Jenkins CI.

## 3. orkis-jobs — 백그라운드 작업 서비스 (58 커밋, 단독에 가까움)

- **회복탄력 Redis 인프라**: ioredis 기반 `RedisConnection`/`RedisManager` — exponential backoff + jitter 재시도(최대 10회), 6종 이벤트 리스너, ping 헬스체크, 다중 커넥션 동시 초기화/graceful disconnect. *(`931a9a9`)*
- **세션 라이프사이클 Job**: `ChatSessionCleanupJob`(타임아웃 강제 완료), `RedisCleanupJob`(완료 세션 5초 간격 삭제), `DataArchiveJob`(세션 데이터를 NAS에 `{chatroom_id}/YYYY-MM-DD.json` 아카이빙).
- **스케줄링 + 외부 트리거 통합**: node-cron `JobScheduler`로 cron·지속·수동 트리거 일원화. 아카이빙을 cron pull → **HTTP push 트리거**(`POST /archive/internal` + 큐잉)로 전환. *(`756162c`, `dc8f616`)*
- **DI 마이그레이션**: 수동 `new`/정적 `DatabaseManager`를 1000줄 이상 제거하고 `@Service`/`@Autowired`/`@InjectConnection` 기반으로 전환, 도메인별 폴더 재편. *(`019bd8a`)*
- **K8s/CI**: Jenkinsfile, deployment/ClusterIP svc, ConfigMap/Secret 주입, NodePool 라벨 정책.

## 4. orkis-desktop — Electron 데스크탑 앱 (82 커밋, 단독 소유)

웹 Orkis를 Electron 39 + electron-vite 데스크탑 앱으로 패키징하고, backend/AI를 **로컬 Podman 컨테이너로 구동하는 런처**.

- **OS 네이티브 통합**: OAuth 인증 창, 플랫폼별 타이틀바(mac hiddenInset), system tray, dmg/pkg·nsis·AppImage 패키징.
- **Podman 컨테이너 런처**: backend/AI를 PyInstaller 바이너리 → Podman 컨테이너 구동으로 전환(`container.service.ts`). *(`c0c2f69`)*
- **Windows 크로스플랫폼**: mac / Windows-NAT / Windows-mirrored 3개 네트워크 모드를 `ContainerNetConfig` 단일 함수로 집중(.wslconfig 자동 감지, WSL 어댑터 IP 추출). *(`98f0fa6`, `e639efb`)*
- **Health 모니터링**: `PodmanEventsMonitor` — `podman events --format json`을 파싱해 `service:ready/unhealthy/down`을 IPC로 push. *(`fb0a2bb`)*
- **부트스트랩 재설계**: `OrkisApplication` 클래스로 라이프사이클 일원화, config/container/ipc/health/services/errors 폴더 응집.

## 5. orkis-desktop-ai — 데스크탑용 Python AI 백엔드 (50 커밋)

데스크탑이 구동하는 AI 추론 서비스(LLM·FAISS 벡터검색·preprocess).

- **FastAPI + uvicorn 도입**: `/chat/start`·`/chat/cancel`·`/preprocess/start` 엔드포인트. HTTP 수신 / 송신 역할 분리. *(`18b1ffa`)*
- **3계층 구조**: controller(HTTP 경계) / service(프로토콜 독립 비즈니스 로직) / dto(Pydantic). *(`f2d6ca5`)*
- **SSE 전환**: TCP socket 전면 제거, `sse_transport.py`(asyncio.Queue) 신규, `StreamingResponse` + `ContextVar` 주입. *(`d2f3bcd`)*
- **비동기 최적화**: 자체 task manager(취소 가능), FAISS 검색 `asyncio.to_thread` 오프로드, schema/data preprocess 병렬화. *(`6f50b54`, `e0ca67f`)*
